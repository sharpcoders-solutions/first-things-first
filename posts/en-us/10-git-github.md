# 💡 What Is Version Control?

Before understanding Git, you need one core concept:

👉 **Version Control = tracking changes over time**

A version control system allows you to:

- Track every change made to your code  
- Go back to previous versions  
- See who changed what  
- Collaborate safely with others  

👉 It keeps the entire history of your project  

---

# ⚙️ What Is Git?

👉 **Git is a version control system**

It is used to:

- Track file changes  
- Manage project history  
- Enable collaboration  

Git is **distributed**, which means:

👉 Every developer has a full copy of the project history  

This makes it:

✅ Fast  
✅ Reliable  
✅ Flexible  

---

# 🔗 What Is GitHub?

👉 **GitHub is a platform built on top of Git**

It allows you to:

- Store code online  
- Share projects  
- Collaborate with others  

👉 Git = tool  
👉 GitHub = platform  

GitHub hosts repositories and adds collaboration features like:

- Code reviews  
- Pull requests  
- Issue tracking  

---

# 🧱 Core Git Concepts

To understand Git, you need a few key ideas:

## 🔹 Repository (Repo)
- The project container  
- Stores all files AND history  

👉 Think: your project + its timeline  

---

## 🔹 Commit
- A snapshot of your project at a moment in time  

👉 Like saving progress  

---

## 🔹 Branch
- A separate version of your code  

👉 Used to develop features safely  

---

## 🔹 Merge
- Combines changes from different branches  

---

## 🔹 Remote Repository
- Online version of your project (e.g., GitHub)  

---

👉 Git works by saving snapshots and history over time   

---

# ⚙️ How Git Works (Simplified)

Basic flow:

1. You change files  
2. You stage changes  
3. You create a commit  
4. Git saves a snapshot  

👉 Over time, this builds a full history of your project  

---

# 🔄 Local vs Remote

## Local (Git)
- Runs on your machine  
- You make changes here  

## Remote (GitHub)
- Stored online  
- Used for collaboration  

👉 You sync using:

- Push → send changes  
- Pull → get changes  

---

# 🏗️ Real-World Example

Imagine you’re building a project:

- Create a feature branch  
- Work without breaking the main code  
- Submit changes for review  
- Merge when ready  

👉 This is how teams collaborate without conflicts

- Working directly on the main branch  
- Ignoring branches
