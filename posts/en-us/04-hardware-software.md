# 🧠 Before Programming: Understand Hardware, Software, and How a Computer Works

⏱️ Reading time: 5 minutes  
✍️ Author: Vitor Santos + 🤖 Copilot

## 🚀 Introduction

When we start learning programming, it’s common to jump straight into code, languages, and frameworks…

But there’s a simple question that few people ask:

👉 **How does code actually work inside a computer?**

Before diving deeper into languages like C#, it’s essential to understand the foundation of everything: **hardware and software**.

---

# 💡 What Is a Computer (Really)

In simple terms, a computer is a machine that:

- Receives data  
- Processes that data  
- Produces a result  

This cycle is known as:

👉 **Input → Processing → Output** 

### Example:
- You type a number (input)  
- The computer calculates  
- It displays the result on the screen (output)  

Simple — yet extremely powerful.

---

# 🧱 What Is Hardware

Hardware is everything you can **see and touch**.

## Examples:
- Keyboard  
- Mouse  
- Monitor  
- RAM  
- Processor (CPU)  
- Storage (HD / SSD)  

👉 It is the physical part of the computer  

💡 Think of it like this:
> Hardware is the **body** of the machine  

It performs actions, but by itself, it doesn’t “know” what to do.

---

# 🧠 What Is Software

Software is the **logical part**, meaning:

- Programs  
- Systems  
- Applications  
- Code  

## Examples:
- Windows / Linux  
- Browser  
- Apps  
- Your C# code  

👉 Software is the set of instructions that tells the hardware what to do   

💡 Simple analogy:
> Software is the **brain**

---

# 🔗 The Relationship Between Hardware and Software

This is the MOST important part:

- Hardware without software = does nothing  
- Software without hardware = cannot exist  

👉 They depend on each other

## Example:
- The keyboard (hardware) sends data  
- The software interprets  
- The processor executes  
- The monitor displays  

Everything works as an integrated system.

---

# 👨‍💻 Where Programming Fits In

Now let’s connect this to what you're learning:

👉 **Programming = creating software**

When you write code:

- You create instructions  
- Those instructions are executed by hardware  
- They produce real-world results  

💡 In other words:
> Your code becomes action inside the machine

---

# 🏗️ What Is Infrastructure (Basic Concept)

Now let’s take one step further…

👉 **IT Infrastructure** is everything that supports systems and applications  

## It includes:
- Hardware  
- Software  
- Networks  
- Storage  

👉 It is the environment where applications run 

💡 Simply put:
> Infrastructure is where everything happens  

---

# 🚀 Why This Matters

Without understanding these concepts, you may:

- Write code without understanding execution  
- Struggle to debug problems  
- Get stuck when learning advanced topics  

But when you understand the fundamentals:

✅ You learn faster  
✅ You solve problems more logically  
✅ You grow as a developer  

---

# 📌 Conclusion

Before focusing only on code, remember:

- Hardware = executes  
- Software = instructs  
- Programming = creates instructions  

👉 Everything in computing starts here.
