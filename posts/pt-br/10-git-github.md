# 💡 O que é Controle de Versão?

Antes de entender Git, você precisa de um conceito fundamental:

👉 **Controle de versão = acompanhar mudanças ao longo do tempo**

Um sistema de controle de versão permite:

- Acompanhar todas as alterações no código  
- Voltar para versões anteriores  
- Saber quem alterou o quê  
- Trabalhar em equipe com segurança  

👉 Ele mantém todo o histórico do projeto

---

# ⚙️ O que é Git?

👉 **Git é um sistema de controle de versão**

Ele é usado para:

- Controlar alterações em arquivos  
- Gerenciar histórico do projeto  
- Facilitar colaboração  

O Git é **distribuído**, o que significa:

👉 Cada desenvolvedor tem uma cópia completa do histórico  

Isso o torna:

✅ Rápido  
✅ Confiável  
✅ Flexível  

---

# 🔗 O que é GitHub?

👉 **GitHub é uma plataforma baseada no Git**

Ele permite:

- Armazenar código na nuvem  
- Compartilhar projetos  
- Trabalhar em equipe  

👉 Git = ferramenta  
👉 GitHub = plataforma  

O GitHub hospeda repositórios e adiciona recursos como:

- Revisão de código  
- Pull requests  
- Controle de tarefas  

---

# 🧱 Conceitos principais do Git

Para entender Git, você precisa conhecer alguns conceitos:

## 🔹 Repositório (Repo)
- Contêiner do projeto  
- Guarda arquivos E histórico  

👉 Pense como: projeto + linha do tempo  

---

## 🔹 Commit
- Um “snapshot” do projeto em um momento específico  

👉 Como salvar progresso  

---

## 🔹 Branch
- Uma versão paralela do código  

👉 Usada para desenvolver funcionalidades com segurança  

---

## 🔹 Merge
- Junta alterações de diferentes branches  

---

## 🔹 Repositório Remoto
- Versão online do projeto (ex: GitHub)  

---

👉 O Git funciona salvando snapshots do projeto ao longo do tempo  

---

# ⚙️ Como o Git funciona (simplificado)

Fluxo básico:

1. Você altera arquivos  
2. Você prepara as mudanças  
3. Você cria um commit  
4. O Git salva o estado  

👉 Com o tempo, isso cria todo o histórico do projeto  

---

# 🔄 Local vs Remoto

## Local (Git)
- Fica na sua máquina  
- Onde você desenvolve  

## Remoto (GitHub)
- Fica na internet  
- Usado para colaboração  

👉 Sincronização:

- Push → enviar mudanças  
- Pull → atualizar código  

---

# 🏗️ Exemplo do mundo real

Em um projeto:

- Você cria uma branch  
- Desenvolve sem afetar o código principal  
- Submete mudanças  
- Faz merge após revisão  

👉 Assim times trabalham sem conflitos  

---

# 🚀 Por que Git & GitHub são importantes?

O desenvolvimento moderno depende deles:

✅ Colaboração segura  
✅ Histórico completo do projeto  
✅ Reversão fácil de mudanças  
✅ Trabalho em equipe eficiente  
✅ Padrão da indústria  

👉 Quase todo desenvolvedor profissional usa Git  

---

# ⚠️ Erros comuns

- Não fazer commits frequentes  
- Trabalhar direto na branch principal  
- Ignorar branches  
- Escrever mensagens de commit ruins  

---

# 📌 Conclusão

- Git = sistema de controle de versão  
- GitHub = plataforma de colaboração  
- Juntos → base do desenvolvimento moderno  

👉 É assim que código real é gerenciado
